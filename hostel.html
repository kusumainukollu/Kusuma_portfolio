Hostel Attendance System

Role: Full Stack Developer • Tech: PHP, MySQL, HTML, CSS, JavaScript

Overview

A web-based attendance and leave management system for hostel students that digitizes manual registers and paper-based workflows. The system enables students and administrators to log attendance, manage leave requests, and view reports from a single interface.

Problem

Hostels often rely on physical registers and handwritten leave forms, which can lead to errors, missing records, and delays in tracking student presence. It is also difficult to quickly understand attendance trends or verify past data.

Solution

I designed and built an online platform where students can view attendance records, submit leave requests, and track approval status, while administrators can mark attendance, review records, approve or reject leaves, and generate summary reports in real time all this happens with a secure login credentials.

My Responsibilities

  • Designed the database schema for Registerations, attendance , leave records and contact queries .
  • Implemented core PHP logic for attendance and leave workflows.
  • Created responsive UI using HTML, CSS, and JavaScript.
  • Handled role-based access for students and admins.
  • Tested flows, fixed bugs, and improved UX based on feedback.

Tech Stack

PHP, MySQL, HTML, CSS, JavaScript

Key Features

Core modules and functionality in the system.

Attendance Management

  • Daily attendance marking for hostel students.
  • View current day and historical attendance records.
  • Highlight absent students and patterns over time.

Leave Workflow & RBAC

  • Students submit leave requests with dates and reasons.
  • Admins approve or reject requests from their dashboard.
  • Role-based access control separates student and admin views.
